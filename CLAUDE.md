## חלק 5 - מה צריך לעשות (סדר עבודה)

### שלב 1 - תמונות אמיתיות (יום 1-2)
**מטרה:** להוסיף תמונה אמיתית אחת לכל אטרקציה, להציג אותה בכרטיס.

**משימות:**
1. צור תיקיית images/ בפרויקט
2. עבור על כל אטרקציה בקובץ, מצא תמונה איכותית בחיפוש, הורד למחשב
3. שמור עם שם הגיוני: pinkertons-bbq.jpg, space-center.jpg וכו'
4. שנה את הקוד כך שיציג את התמונה בכרטיס (מעל הכותרת)
5. בקרטיס מקופל - תמונה קטנה כתצוגה מקדימה
6. בכרטיס פתוח - תמונה גדולה יותר
7. דחוף לגיטהאב, בדוק שעובד

### שלב 2 - מצב "היום" (יום 2)
**מטרה:** כשהטיול יתחיל, פתיחת החוברת תפתח אוטומטית את היום הנוכחי.

**משימות:**
1. לכל יום בקוד הגדר תאריך
2. בטעינה: בדוק תאריך נוכחי
3. אם זה אחד מימי הטיול - פתח את היום הזה
4. אם לא - פתח כברירת מחדל יום 1
5. הוסף סימן ויזואלי "היום" על היום הנוכחי

### שלב 3 - מסך פתיחה אישי (יום 2-3)
**מטרה:** מסך טעינה של 2-3 שניות עם ספירה לאחור.

**משימות:**
1. הוסף div שיוצג בכניסה
2. עיצוב: רקע גרדיאנט חם, אנימציית fade-in
3. טקסט: "שלום דר! עוד X ימים ל-NRG Stadium"
4. חישוב X דינמי לפי תאריך נוכחי
5. אחרי 2.5 שניות - fade out והחוברת מופיעה
6. אפשרות "דלג" קטנה בפינה

### שלב 4 - אנימציית כדור פותח (יום 3)
**מטרה:** כדור כדורגל מתגלגל ויוצר את הכותרת.

**משימות:**
1. תוסף SVG של כדור כדורגל
2. אנימציית CSS: גלגול מצד שמאל למרכז
3. אחרי הגלגול - הכדור "מתפוצץ" והכותרת מופיעה
4. אורך כולל: 1.5 שניות
5. רק בכניסה הראשונה, לא בכל ניווט

### שלב 5 - תחזית מזג אוויר חיה (יום 4)
**מטרה:** מזג אוויר אמיתי לכל יום.

**משימות:**
1. רישום ל-OpenWeatherMap (חינמי, יש להירשם)
2. קבל API key
3. שמור אותו בקובץ נפרד config.js
4. הוסף API key ל-.gitignore כדי שלא יעלה לגיטהאב
5. לכל יום קרא תחזית לפי תאריך וקואורדינטות העיר
6. הצג ליד כותרת היום: אייקון מזג אוויר, טמפרטורה, סיכויי גשם
7. אם זה היום הנוכחי - גם תחזית שעתית

### שלב 6 - תמונה כרקע חי (יום 4-5)
**מטרה:** רקע מטושטש של תמונת העיר בכל לשונית.

**משימות:**
1. מצא 4 תמונות איכותיות של 4 הערים (סקיילין, נוף עיר)
2. שמור ב-images/backgrounds/
3. שנה את הרקע של כל לשונית לפי העיר
4. הוסף שכבת blur ו-opacity להמשכת קריאות
5. עם אנימציית מעבר כשמחליפים לשונית

### שלב 7 - אפקטים ויזואליים לכל יום (יום 5-6)
**מטרה:** סמלים שמתעופפים ברקע, ייחודיים לכל יום.

**משימות:**
1. צור library קטן של אייקונים SVG (כדור, דגלים, אצטדיון, פסל החירות)
2. לכל יום הגדר 2-3 סמלים מתאימים
3. אנימציית CSS: צף לאט ברקע, opacity נמוכה (10-15%)
4. וריאציות מהירות ומיקומים, כדי שלא יראה מונוטוני
5. במצב נייד: צמצם כמות לביצועים

### שלב 8 - מפת הטיול הגדולה (יום 6)
**מטרה:** עמוד מיוחד עם מפה ענקית של ארה"ב + קו המסע.

**משימות:**
1. הוסף לשונית/מקטע חדש: "מפת המסע"
2. מפת ארה"ב באמצעות Leaflet עם תצוגה כללית
3. סיכות גדולות ב-4 הערים
4. קווי SVG מצוירים ביד שמחברים את הערים בסדר
5. אנימציית "ציור" - הקו נמשך לאט מיוסטון לניו יורק
6. בכל סיכה: שם העיר, תאריכי שהיה, מספר אטרקציות

### שלב 9 - מסלול היום אוטומטי (יום 6-7)
**מטרה:** המערכת מציעה מסלול אופטימלי לפי בחירות.

**משימות:**
1. הוסף כפתור "סמן כמעניין" לכל אטרקציה
2. שמור בחירות ב-LocalStorage
3. כפתור "צור לי מסלול ליום"
4. לוגיקה: 
   - לקח האטרקציות שסומנו ביום
   - חשב מרחקים ביניהן
   - סדר אותן באלגוריתם פשוט (Nearest Neighbor)
   - הצג את הסדר עם זמני נסיעה משוערים
5. תצוגה: רשימה ממוספרת עם חיצים בין הצעדים

### שלב 10 - צ'אט עם דר-בוט (יום 7+)
**מטרה:** עוזר אישי שיודע הכל על הטיול.

**משימות:**
1. רישום ל-Claude API
2. שמירת API key בצורה מאובטחת
3. בנה ממשק צ'אט קטן בפינת המסך
4. שלח עם כל הודעה את כל המידע על החוברת כקונטקסט
5. הנחיית מערכת: "אתה עוזר טיולים אישי של דר..."
6. אפשרויות מהירות: "מה כדאי לי היום?", "איפה הכי קרוב לאכול?"

### שלב 11 - גיבוי אופליין (שבוע הבא)
**מטרה:** החוברת תעבוד בלי אינטרנט.

**משימות:**
1. צור Service Worker
2. הגדר אסטרטגיית cache: Cache First
3. רשום את כל הקבצים, התמונות, הספריות
4. הוסף manifest.json - אפשרות "התקן כאפליקציה"
5. בדוק בכל סוגי המכשירים
